Deep dive into the SQL SELECT Statement

Structure of a SELECT Statement

Here is an example SELECT statement.

SELECT * FROM highscores WHERE score > 50

It returns this data:
firstname lastname score
Eve Jackson 94
Bob Doe 100

The SELECT statement is made up of 3 parts.

  1. SELECT x
  2. FROM y
  3. WHERE z

What are x, y, and z?

  • x: The name of the column you want to see (or * for all columns). Each column holds a certain type of data.

    • x

      x can be extended using a comma to show multiple columns.

      Here's an example:

      SELECT firstname, lastname FROM highscores WHERE score > 50

      firstname lastname
      Jill Smith
      Eve Jackson
      Bob Doe
  • y: The name of the table you're drawing from
  • z: Called the Clause, z reduces the amount of data you pull to a more specific subset. It's like when you "Refine your search results." It is actually optional.

y

y can be extended to allow you to draw data from multiple tables using JOIN statements. We'll expand on those later. For now, we'll work with drawing data from only one table.

z

z, called the Clause, reduces the amount of data you pull to a more specific subset. It's like when you "Refine your search results"

It is made up of three parts:

  • a column name,

    • In the same format as x, but may be a different column from the one you

  • operator,
      • List of Operators
    • = Equal
    • <> Not equal. Note: In some versions of SQL this operator may be written as !=
    • > Greater than
    • < Less than
    • >= Greater than or equal
    • <= Less than or equal
    • BETWEEN Between an inclusive range
    • LIKE Search for a pattern
    • IN To specify multiple possible values for a column
  • and value.

    • The value is what you want the data in the column you've chosen to match. For example, the number 50 in the score column, or the string "smith" in the lastname column for all players who have that lastname.
